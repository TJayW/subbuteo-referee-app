# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please follow these steps:

1. **DO NOT** open a public issue
2. Email security concerns to the repository owner (check GitHub profile)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

You should receive a response within 48 hours. We will work with you to understand and address the issue promptly.

## Security Best Practices

This application:
- Runs entirely client-side (no backend)
- Does not transmit data to external servers
- Uses localStorage for persistence (client-side only)
- Does not collect analytics or telemetry

**User Responsibilities:**
- Keep your browser up to date
- Be cautious when importing match data from untrusted sources
- Clear browser data when using shared computers
